This whole game is one GGUF

  
A first-person maze with a monster that chases you (real BFS
     pathfinding, tabulated in the weights), hitscan shooting and hit
     points — all inside a 294 MB GGUF served by a stock ollama. The
     frontend sends only the pressed key.

  
The state is re-serialized every tick: the prompt carries position,
     angle, enemy, HP and tick counter, and the model answers with the next
     state plus the rendered frame (32 raycast wall columns and a
     sprite token). The joint state space is too big for one lookup table,
     so the transformer uses its own generated tokens as registers: one
     tick is a pipeline of small tables (relative position → line of
     sight → hit → enemy move → damage → sprite), each
     step reading the tokens the previous one just wrote.

  
No training: every weight is constructed, and the generator refuses to
     write the file unless it replays perfectly against a reference
     implementation.
