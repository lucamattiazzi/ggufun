The bubble has burst. The venture capital is gone, the APIs are dark, and millions of ollama servers are still humming in basements with nothing to do. This catalog is a salvage operation: every machine on the floor is a hand-welded GGUF — no training, no dataset, just weights forged one tensor at a time until a stock inference server does something it was never meant to do.

How this particular page reached you

It was not read from a disk. It was decoded. The whole file — the doctype, the stylesheet above, this sentence, the closing tag — is a single token in the model's vocabulary. Your request was a token too: GET /about.html; matches one entry, its embedding names one state, and the unembedding turns that state into exactly one other token: this page. Then end-of-sequence. Two tokens, one page, no arithmetic.

And not just this page: the entire catalog is in here. The frameset, the banner, the machine plates, the stylesheet, the very ledger of machines the plates are drawn from — each one its own token, each one its own GET. Even the background tile is in the weights, base64'd into the stylesheet, because a token is a UTF-8 string and a PNG is not. What you are browsing is a picture of a website, drawn from memory, correctly, every time.

Ask for a path that was never compiled in and nothing matches: the request crumbles into individual byte tokens, every one of them votes for the same default, and you get the 404. The machine does not check whether a file exists. It has no files, and no filesystem to check.

An honest note on the size

The badge is not a boast, it is a confession. The site this model serves weighs about two hundred kilobytes (the logo lives in here too, in base64); the model weighs three hundred and seventy-one. Everything a real web server would keep on disk is kept here in a vocabulary, plus the ballast the format demands: 256 byte tokens, a position table we never read, two matrices. The compiler prints the exact figure every time it runs, and refuses to be coy about it:

compression: 1:1.9, in the wrong direction

That number is the product, and watch which way it moves: when this model held only two pages it was 1:14, and the whole catalog, logo included, brought it to 1:1.9. The ballast is fixed and the transcription is not, so the more you put in, the closer it creeps to 1:1 — from above, always, and it never gets there. This machine does not compress. It transcribes, and then pays to carry what it transcribed.
